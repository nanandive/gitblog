코딩테스트 공부하면서 가장 어렵게 느꼈던 게 DP였다.

뭐가 어려운지 분석해보니: "어떤 문제에 DP를 쓸지 알 수가 없다"였다. 정렬은 정렬 조건이 있으면 쓰고, BFS는 최단 거리면 쓰고, 이런 식으로 명확한데 DP는 언제 써야 하는지가 모호했다.

## DP가 왜 어려운가

DP(Dynamic Programming)는 **접근 방식**이다. 특정 문제 유형이 아니다.

정렬 알고리즘은 "어떻게 정렬할 것인가"다. DP는 "어떻게 문제를 쪼갤 것인가"다. 쪼개는 방법을 직접 발견해야 하기 때문에 어렵다.

그리고 DP를 공부할 때 "피보나치 수열" 예제를 제일 먼저 봤는데, 실제 코딩테스트 DP 문제는 피보나치와 다른 경우가 많다.

## DP를 쓸 수 있는 조건

두 가지:
1. **최적 부분 구조**: 전체 최적해가 부분 최적해로 구성된다
2. **중복 부분 문제**: 동일한 부분 문제가 반복 등장한다

이걸 이론으로 외우면 의미가 없다. "관계식이 존재하는가?"로 생각하는 게 실용적이다.

## 세 단계 접근

**동전 문제**: 1, 3, 4짜리 동전으로 N원을 만들 때 최소 동전 개수

### 1단계: 재귀적 구조 파악

N원을 만들기 직전 상태를 생각한다.

- (N-1)원에서 1짜리 동전 추가
- (N-3)원에서 3짜리 동전 추가
- (N-4)원에서 4짜리 동전 추가

이 세 가지 경우만 존재한다. 재귀적 구조를 찾았다.

### 2단계: 점화식 수립

F(N) = (N원을 만들 때 최소 동전 개수)

```
F(N) = min(F(N-1), F(N-3), F(N-4)) + 1
```

### 3단계: 초기값 처리

N이 4 이상이면 N-1, N-3, N-4가 모두 양수라서 괜찮다. 하지만 N이 1, 2, 3이면 음수 인덱스가 생긴다.

```python
def min_coins(n):
    f = [0] * (n + 1)
    
    # 초기값 처리 (음수 인덱스 방지)
    f[1] = 1  # 1 = 1
    f[2] = 2  # 2 = 1+1
    f[3] = 1  # 3 = 3
    f[4] = 1  # 4 = 4
    
    for i in range(5, n + 1):
        f[i] = min(f[i-1], f[i-3], f[i-4]) + 1
    
    return f[n]
```

시간복잡도: O(N). 재귀로 풀면 지수 시간이지만 메모이제이션으로 O(N)이 된다.

## 메모이제이션

DP의 핵심이다.

F(6)을 구하기 위해 F(5), F(3), F(2)를 구해야 한다. F(5)를 구하면서 F(4), F(2), F(1)을 구한다. 여기서 F(2)가 중복으로 계산된다.

메모이제이션은 한 번 계산한 결과를 저장해두고 재사용한다. DP 테이블(`f` 배열)이 바로 이 저장 공간이다.

## 실전 DP 문제 접근

```python
# 1. 문제를 보고 "지금 상태가 이전 어떤 상태에서 왔나?" 생각
# 2. 관계식을 수식으로 써본다
# 3. 초기값(언제 관계식이 성립 안 하는가)을 찾는다
# 4. 코드로 구현
```

관계식이 안 보이면:
- 문제를 다르게 해석해본다
- 상태 정의를 바꿔본다 (1차원 DP → 2차원 DP)
- DP가 아닐 수도 있다 (그리디, BFS 등으로 접근)

## 자주 나오는 DP 유형

```
배낭 문제 (Knapsack): 무게 제한 내 최대 가치
  → 상태: dp[i][w] = i번째 아이템까지 고려, 무게 w일 때 최대 가치

최장 공통 부분수열 (LCS):
  → 상태: dp[i][j] = s1[0..i], s2[0..j]의 LCS 길이

최장 증가 부분수열 (LIS):
  → 상태: dp[i] = i번째 원소로 끝나는 최장 증가 부분수열 길이
```

## 회고

DP 문제를 처음 보면 아직도 막막하다. 하지만 "직전 상태를 어떻게 정의하는가"라고 질문을 바꾸고 나서부터 시작점이 생겼다.

세 단계를 번거롭더라도 지키는 것이 중요하다. 점화식을 머릿속으로만 생각하고 바로 구현하려다가 초기값 처리에서 막히는 경우가 많았다.

관계식 찾는 능력은 많이 풀어봐야 는다. 다른 알고리즘보다 연습 볼륨이 더 필요하다.
