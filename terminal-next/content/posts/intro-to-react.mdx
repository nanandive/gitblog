# Modern React with TypeScript

React has evolved significantly. With the introduction of Hooks and now Server Components, the landscape is more exciting than ever.

## Why TypeScript?

TypeScript provides type safety that catches errors during development rather than at runtime. For large-scale React applications, it's not just a preference; it's a requirement for maintainability.

## Key Takeaways

1. **Functional Components are the standard.** Class components are legacy now.
2. **Use `useMemo` and `useCallback` judiciously.** Premature optimization is the root of all evil.
3. **Strict mode is your friend.** It helps catch common bugs during development.

```typescript
// Example: Type-safe custom hook
function useLocalStorage<T>(key: string, initialValue: T) {
  const [stored, setStored] = useState<T>(() => {
    if (typeof window === 'undefined') return initialValue;
    try {
      const item = window.localStorage.getItem(key);
      return item ? JSON.parse(item) : initialValue;
    } catch {
      return initialValue;
    }
  });
  
  const setValue = (value: T) => {
    setStored(value);
    window.localStorage.setItem(key, JSON.stringify(value));
  };
  
  return [stored, setValue] as const;
}
```

## Conclusion

TypeScript + React is now the industry standard. If you're not using it yet, it's time to make the switch.
